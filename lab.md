根据SYSEN 5381课程的课堂录音内容，以下是关于本次Lab（实验）的提交要求和资料位置的详细说明：

1. 提交要求 (Submission Requirements)

为了获得学分，你需要在 Canvas/课程平台 上提交以下 三样东西：

完整的脚本代码 (Complete Script): 你编写的用来运行整个流程的代码。

最终报告的截图 (Screenshot): 展示由 AI 生成的最终报告的截图。

简要说明 (Description): 简单描述一下你做了什么（比如你用了什么 API，让 AI 做了什么处理）。

Lab 的核心任务目标：
你需要制作一个“记者软件 (Reporter Software)”，其基本工作流如下：

获取数据： 使用 API query 获取数据（可以是真实的 API，也可以是模拟的数据）。

AI 处理： 将获取的数据喂给 AI 模型。

生成输出并保存： 让 AI 生成一份“报告”，并将这份报告写入到一个文件中（Write to file）。

文件格式不限： 教授明确表示（23:46），生成的文件可以是 CSV, TXT, PDF, HTML, DOC 等任何格式，关键是要完成“写入文件”这一步。

2. 需要看的资料在哪里 (Where to find materials)

本次 Lab 的具体指南和相关资料位于课程平台的 Module 3 中。

重要提示（如果发现资料打不开）：
录音中教授特别提到（22:18），很多同学遇到 Module 3 变成灰色（不可用） 的情况。

原因： 这是因为课程设置了“先修要求”。你必须先完成 "Getting Started" 模块中的所有项目。

解决方法： 回去检查 "Getting Started" 部分，确保每一个条目都点过了（特别是 Discussion tab），直到上面的红色标记变成 绿色的对勾 (Green Checkmark)。一旦完成这些，Module 3 就会解锁，你就能看到 Lab 的详细文档了。

总结建议

这次 Lab 可以作为你 Homework 2 的一次预演。教授建议你利用这次机会尝试 API 到 AI 再到文件输出的全过程。如果你还没想好用什么 API，甚至可以先手动编造一些数据（Fake data），重点是跑通这个 AI 处理和文件生成的流程。
